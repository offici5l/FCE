
# كيفية اختبار الخدمة محليًا باستخدام Docker

هذا الدليل يشرح كيفية بناء وتشغيل خدمة FCE على جهازك المحلي باستخدام Docker، ثم اختبارها باستخدام الواجهة الأمامية البسيطة.

تأكد من أنك تقوم بتشغيل هذه الأوامر من داخل مجلد `test`.

### الخطوة 1: بناء حاوية Docker (Build)

الخطوة الأولى هي بناء "صورة" (Image) للحاوية من الـ `Dockerfile` المصحح. هذه العملية تتم مرة واحدة (أو في كل مرة تعدل فيها الملفات).

```bash
# ادخل إلى مجلد test أولاً
cd test

# قم ببناء الصورة وأعطها اسمًا مثل fce-local
docker build -t fce-local .
```

### الخطوة 2: تشغيل حاوية Docker (Run)

بعد بناء الصورة، يمكنك الآن تشغيلها كحاوية (Container). هذا الأمر سيبدأ خادم الويب الخاص بك.

```bash
# ملاحظة: إذا كنت لا تزال داخل مجلد test، يمكنك تشغيل هذا الأمر مباشرة
docker run --rm -p 127.0.0.1:8080:8080 fce-local
```

**شرح الأمر:**
- `docker run`: أمر تشغيل الحاوية.
- `--rm`: يقوم بحذف الحاوية تلقائيًا بعد إيقافها (للحفاظ على النظافة).
- `-p 127.0.0.1:8080:8080`: هذا الجزء مهم جدًا. يقوم بربط المنفذ `8080` على جهازك المحلي بالمنفذ `8080` داخل الحاوية. هذا ما يسمح للمتصفح بالتواصل مع الخادم.
- `fce-local`: اسم الصورة التي قمنا ببنائها في الخطوة الأولى.

**متابعة التقدم:** ستعرض هذه الطرفية سجلات الخادم مباشرة. يمكنك رؤية طلبات التنزيل والاستخراج هنا، وهو أمر مفيد جدًا لمتابعة ما يحدث.

### الخطوة 3: استخدام موقع الاختبار المحلي

1.  **لا تغلق الطرفية** التي تعمل بها حاوية Docker.
2.  افتح متصفح الويب على جهازك.
3.  اذهب إلى مجلد المشروع، ثم مجلد `test`، ثم `frontend`، وافتح ملف `index.html` مباشرة في المتصفح.
    - يمكنك عادةً سحب وإفلات الملف في نافذة المتصفح، أو استخدام قائمة `File > Open File`.
4.  الآن يمكنك استخدام النموذج الموجود في الصفحة لإرسال طلب إلى خادمك المحلي.

عندما تضغط على "Start Extraction"، سيقوم الموقع بإرسال الطلب إلى `http://127.0.0.1:8080`، وستبدأ العملية في نافذة Docker الطرفية. عند اكتمالها بنجاح، سيقوم المتصفح تلقائيًا بتنزيل الملف الناتج.
